# G2b — @PermuteReturn Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable return type narrowing by arity — `Step${i}` methods can return `Step${i+1}<T1,...,T${i+1}>`, with the last class in the range automatically omitting the method (boundary omission). Supports both zero-annotation implicit inference (Maven plugin inline mode) and explicit `@PermuteReturn` (APT mode and `alpha(j)` naming).

**Architecture:** Three independent pieces that build on each other: (1) `@PermuteReturn` annotation; (2) APT path — read `@PermuteReturn` via JavaParser in `PermuteProcessor.generatePermutation()`, replace the `Object` sentinel return type, omit boundary methods; (3) InlineGenerator path — scan the parent CU for all generated class names, apply implicit inference (Condition 1+2) plus parameter type inference, then handle explicit `@PermuteReturn` as an override. The key infrastructure — the generated-class-name set — is built once per template CU and reused across all permutations.

**Tech Stack:** JavaParser (AST manipulation), Google compile-testing (APT tests), JUnit 4, N4 expression functions (alpha, lower, typeArgList — already implemented).

**Depends on:** G2a must be merged first (for `@PermuteDeclr` on method params used in APT mode templates).

```java
package io.quarkiverse.permuplate;

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

/**
 * Explicitly controls the return type of a method for each permutation value.
 *
 * <p><b>Not needed in the common case</b> (inline/Maven plugin mode with {@code T${j}} naming):
 * when the return type is a generated class whose type arguments follow the {@code T${j}}
 * naming convention, the return type and any undeclared parameter types are inferred
 * automatically — no annotation required.
 *
 * <p>Use {@code @PermuteReturn} when:
 * <ul>
 * <li><b>APT mode</b> — the template must compile; use {@code Object} as the sentinel return
 *     type and {@code @PermuteReturn} to specify the actual return type</li>
 * <li><b>Custom naming ({@code alpha(j)})</b> — inference requires {@code T${j}} convention;
 *     single-letter names do not trigger it</li>
 * <li><b>Non-linear offset</b> — inference assumes {@code i+1}; use explicit for other offsets</li>
 * <li><b>Mixed fixed+growing type args</b> — use {@code typeArgs} with {@code typeArgList()}</li>
 * <li><b>Override boundary omission</b> — use {@code when="true"} to force generation of the
 *     method even when the evaluated class name is not in the generated set</li>
 * </ul>
 *
 * <p><b>Boundary omission:</b> by default, when the evaluated {@code className} is not in the
 * set of class names generated by {@code @Permute} in the current template, the method is
 * silently omitted from that generated class — exactly mirroring the hand-written pattern
 * where the last class in a chain has no {@code join()} method.
 *
 * <p>APT mode example:
 * <pre>{@code
 * @Permute(varName="i", from=1, to=4, className="Step${i}")
 * public class Step1<T1> {
 *     @PermuteReturn(className="Step${i+1}",
 *                    typeArgVarName="j", typeArgFrom="1", typeArgTo="${i+1}", typeArgName="T${j}")
 *     public Object join(@PermuteDeclr(type="Source<T${i+1}>") Object src) { return null; }
 * }
 * // Step4.join() is omitted — Step5 is not in the generated set
 * }</pre>
 */
@Retention(RetentionPolicy.SOURCE)
@Target(ElementType.METHOD)
public @interface PermuteReturn {

    /**
     * Template expression for the return type. Typically a class name
     * (e.g., {@code "Step${i+1}"}), but may also be a type variable name
     * (e.g., {@code "${alpha(i)}"} to return the i-th type parameter).
     * Evaluated against the outer permutation context for each generated class.
     */
    String className();

    /**
     * Variable name for the type argument expansion loop (e.g., {@code "j"}).
     * When empty (the default), no type arguments are generated (raw return type).
     * Mutually exclusive with {@code typeArgs} — set one or the other, not both.
     */
    String typeArgVarName() default "";

    /** Lower bound of the type argument loop. Default {@code "1"}. */
    String typeArgFrom() default "1";

    /**
     * Upper bound of the type argument loop (e.g., {@code "${i+1}"}).
     * Required when {@code typeArgVarName} is set.
     */
    String typeArgTo() default "";

    /** Type argument name template per loop value (e.g., {@code "T${j}"}). */
    String typeArgName() default "";

    /**
     * Full type argument list as a JEXL expression (e.g.,
     * {@code "DS, ${typeArgList(1, i, 'T')}"}). Use when type args are a mix
     * of fixed and growing values. Overrides the {@code typeArgVarName} loop.
     * Mutually exclusive with {@code typeArgVarName}.
     */
    String typeArgs() default "";

    /**
     * Optional JEXL guard expression. When empty (default), boundary omission
     * applies: the method is omitted when the evaluated {@code className} is not
     * in the generated set. Set to {@code "true"} to force generation regardless.
     */
    String when() default "";
}
```

    /** Helper: parse a source string and call InlineGenerator on the named template class. */
    private static String generateInline(String source, String templateClassName,
            String varName, int from, int to, String classNameTemplate, int forI) {
        CompilationUnit cu = StaticJavaParser.parse(source);
        ClassOrInterfaceDeclaration template = cu.findFirst(
                ClassOrInterfaceDeclaration.class,
                c -> c.getNameAsString().equals(templateClassName)).orElseThrow();
        PermuteConfig config = new PermuteConfig(varName, from, to, classNameTemplate,
                new String[0], new PermuteVarConfig[0], true, false);
        CompilationUnit output = InlineGenerator.generate(cu, template,
                config, List.of(Map.of(varName, forI)));
        return output.toString();
    }

    /**
     * Processes @PermuteReturn annotations on methods:
     * - Evaluates the return type class name and type arguments
     * - Applies boundary omission (omits the method if the evaluated class name is not
     *   in the generated set, unless when="true")
     * - Replaces the sentinel return type (Object) with the computed return type
     * - Removes the @PermuteReturn annotation from the method
     */
    private void applyPermuteReturn(ClassOrInterfaceDeclaration classDecl,
            EvaluationContext ctx,
            Set<String> generatedSet,
            TypeElement element) {

        List<MethodDeclaration> toRemove = new ArrayList<>();

        classDecl.getMethods().forEach(method -> {
            // Find @PermuteReturn annotation
            Optional<NormalAnnotationExpr> annOpt = method.getAnnotations().stream()
                    .filter(a -> {
                        String n = a.getNameAsString();
                        return (n.equals("PermuteReturn") || n.equals("io.quarkiverse.permuplate.PermuteReturn"))
                                && a instanceof NormalAnnotationExpr;
                    })
                    .map(a -> (NormalAnnotationExpr) a)
                    .findFirst();

            if (annOpt.isEmpty()) return;

            NormalAnnotationExpr ann = annOpt.get();

            // V2: typeArgVarName requires typeArgTo and typeArgName
            String typeArgVarName = getAnnAttr(ann, "typeArgVarName");
            String typeArgTo = getAnnAttr(ann, "typeArgTo");
            String typeArgName = getAnnAttr(ann, "typeArgName");
            String typeArgs = getAnnAttr(ann, "typeArgs");

            if (typeArgVarName != null && !typeArgVarName.isEmpty()) {
                if (typeArgs != null && !typeArgs.isEmpty()) {

                            element);

            String classNameTemplate = getAnnAttr(ann, "className");
            if (classNameTemplate == null || classNameTemplate.isEmpty()) return;

            String evaluatedClassName;
            try {
                evaluatedClassName = ctx.evaluate(classNameTemplate);
            } catch (Exception e) {
                processingEnv.getMessager().printMessage(Diagnostic.Kind.ERROR,
                        "@PermuteReturn: cannot evaluate className \"" + classNameTemplate + "\": " + e.getMessage(),
                        element);
                return;
            }

            // V3: typeArgFrom > typeArgTo
            String typeArgFrom = getAnnAttr(ann, "typeArgFrom");
            if (typeArgVarName != null && !typeArgVarName.isEmpty() && typeArgFrom != null && typeArgTo != null && !typeArgTo.isEmpty()) {
                try {
                    int fromVal = ctx.evaluateInt(typeArgFrom.isEmpty() ? "1" : typeArgFrom);
                    int toVal = ctx.evaluateInt(typeArgTo);
                    if (fromVal > toVal) {
                        processingEnv.getMessager().printMessage(Diagnostic.Kind.ERROR,
                                "@PermuteReturn has invalid type argument range: from=" + fromVal + " is greater than to=" + toVal,
                                element);
                        return;
                    }
                } catch (Exception ignored) {
                }
            }

            // Boundary omission check
            String whenExpr = getAnnAttr(ann, "when");
            boolean shouldGenerate;
            if (whenExpr == null || whenExpr.isEmpty()) {
                shouldGenerate = generatedSet.contains(evaluatedClassName);
            } else {
                try {
                    String evaluated = ctx.evaluate("${" + whenExpr + "}");
                    shouldGenerate = Boolean.parseBoolean(evaluated);
                } catch (Exception e) {
                    shouldGenerate = generatedSet.contains(evaluatedClassName);
                }
            }

            if (!shouldGenerate) {
                toRemove.add(method);
                return;
            }

            // Build return type string
            String returnTypeStr = buildReturnTypeStr(evaluatedClassName, ann, ctx);

            // Replace the method return type
            try {
                method.setType(StaticJavaParser.parseType(returnTypeStr));
            } catch (Exception e) {
                processingEnv.getMessager().printMessage(Diagnostic.Kind.ERROR,
                        "@PermuteReturn: cannot parse computed return type \"" + returnTypeStr + "\": " + e.getMessage(),
                        element);
                return;
            }

            // Remove @PermuteReturn annotation
            method.getAnnotations().removeIf(a -> a == ann);
        });

        // Remove boundary-omitted methods from the class
        toRemove.forEach(m -> classDecl.getMembers().removeIf(member -> member == m));
    }

    /** Builds the full return type string including type arguments. */
    private String buildReturnTypeStr(String className, NormalAnnotationExpr ann, EvaluationContext ctx) {
        String typeArgs = getAnnAttr(ann, "typeArgs");
        if (typeArgs != null && !typeArgs.isEmpty()) {
            // typeArgs is a full JEXL expression for the complete type arg list
            String evaluatedArgs = ctx.evaluate("${" + typeArgs + "}");
            return evaluatedArgs.isEmpty() ? className : className + "<" + evaluatedArgs + ">";

        String typeArgVarName = getAnnAttr(ann, "typeArgVarName");
        if (typeArgVarName == null || typeArgVarName.isEmpty()) {
            return className; // raw return type
        }

        String typeArgFrom = getAnnAttr(ann, "typeArgFrom");
        String typeArgTo = getAnnAttr(ann, "typeArgTo");
        String typeArgName = getAnnAttr(ann, "typeArgName");
        if (typeArgFrom == null || typeArgFrom.isEmpty()) typeArgFrom = "1";

        try {

            int to = ctx.evaluateInt(typeArgTo);

            for (int j = from; j <= to; j++) {

                sb.append(ctx.withVariable(typeArgVarName, j).evaluate(typeArgName));
            }
            sb.append(">");
            return sb.toString();

            return className;
        }

    /**
     * Applies implicit return type inference (G2 Mechanism 1) to all methods in the
     * generated class that do NOT have an explicit @PermuteReturn annotation.
     *
     * <p>Inference fires when BOTH conditions hold:
     * <ol>
     * <li>The return type base class name is in {@code allGeneratedNames}</li>
     * <li>The return type's type arguments consist of declared type params (fixed)
     *     followed by undeclared type variables matching {@code T+number} (growing tip)</li>
     * </ol>
     *
     * <p>When inference fires, parameter types that reference the same undeclared
     * type variables are also updated (parameter type inference).
     *
     * <p>Methods whose inferred return type class is not in {@code allGeneratedNames}
     * are omitted (boundary omission).
     */
    private static void applyImplicitInference(ClassOrInterfaceDeclaration classDecl,
            EvaluationContext ctx,
            Set<String> allGeneratedNames) {

        // Collect declared class type parameter names (e.g. "T1", "R")
        Set<String> declaredTypeParams = new HashSet<>();
        classDecl.getTypeParameters().forEach(tp -> declaredTypeParams.add(tp.getNameAsString()));

        List<MethodDeclaration> toRemove = new ArrayList<>();

        classDecl.getMethods().forEach(method -> {
            // Skip methods with explicit @PermuteReturn (handled by applyPermuteReturn)
            boolean hasExplicit = method.getAnnotations().stream().anyMatch(a -> {
                String n = a.getNameAsString();
                return n.equals("PermuteReturn") || n.equals("io.quarkiverse.permuplate.PermuteReturn");
            });
            if (hasExplicit) return;

            // Get return type
            String returnTypeStr = method.getTypeAsString();
            if (returnTypeStr.equals("void") || returnTypeStr.equals("Object")) return;

            // Parse the return type to extract base class name and type arguments
            ReturnTypeInfo info = parseReturnTypeInfo(returnTypeStr);
            if (info == null) return;

            // Condition 1: base class in generated set?
            if (!allGeneratedNames.contains(info.baseClass())) return;

            // Condition 2: type args — fixed declared params followed by undeclared T+number vars
            // The "growing tip" is the set of undeclared T+number type vars
            List<String> fixedArgs = new ArrayList<>();
            List<String> growingTip = new ArrayList<>();
            boolean conditionTwoHolds = true;
            boolean passedFixed = false;

            for (String arg : info.typeArgs()) {
                if (declaredTypeParams.contains(arg)) {
                    if (passedFixed) { conditionTwoHolds = false; break; } // fixed must come first
                    fixedArgs.add(arg);
                } else if (isTNumberVar(arg)) {
                    passedFixed = true;
                    growingTip.add(arg);
                } else {
                    conditionTwoHolds = false;
                    break;
                }
            }

            if (!conditionTwoHolds || growingTip.isEmpty()) return;

            // Determine offset and compute new return type for this permutation
            // The template showed Step2 in Step1 — offset is arity_of_return - arity_of_template
            // We infer: return class = baseClass with numeric suffix = (current i-value + offset)
            // Simpler: infer offset from the template's growing tip size at template base
            // For each permutation, grow the tip by (i - templateBaseI) more entries

            // The growing tip at the template (base case) has growingTip.size() elements.
            // For the current permutation, we need to re-evaluate: how many tip elements?
            // The tip always ends at T${i+1} where i is the current outer variable value.
            // We detect this by the last element of the tip: it's T(n) where n = i_template + 1
            // For current i, tip grows to T(i+1).

            // Find the suffix number of the last growing tip element
            int lastTipNum = tNumber(growingTip.get(growingTip.size() - 1));
            // Find the class arity offset: offset = number of declared class type params
            // (e.g., Step1<T1> has 1 declared type param; return type Step2<T1,T2> has tip T2)
            // At current permutation i, the number of declared type params = declaredTypeParams.size()
            // The new tip should go from T(declaredTypeParams.size()+1) to T(declaredTypeParams.size()+1) again
            // Actually: the tip at i=1 is [T2]; at i=2 should be [T2,T3]; etc.
            // Formula: new tip = T(declaredFixed+1) .. T(declaredFixed + i_offset)
            //   where i_offset = tip size at current i = lastTipNum (at template) - declaredFixed + (i - i_template)
            // But we don't know i_template here. Use simpler heuristic:
            // At template: tip = [T(lastTipNum)]; growing tip size = 1.
            // At current i: tip size = 1 + (currentFixedCount - templateFixedCount), but we don't track that.

            // SIMPLER APPROACH: directly compute.
            // The tip grows by 1 per outer permutation beyond the template.
            // The template's first undeclared element is T(n) where n = declaredTypeParams.size() + 1.
            // After PermuteTypeParamTransformer, the declared type params list now reflects the current i.
            // declaredTypeParams.size() == current arity (i).
            // At current i: the new tip should be T(i+1) only (one new element).
            // New return type = baseClass<fixedArgs..., T(i+1)>
            // And new class name: baseClass with suffix = (current suffix from template) + (i - template_i)

            // Determine the base class suffix number (to compute offset)
            int baseClassSuffix = classNameSuffix(info.baseClass());
            if (baseClassSuffix < 0) return; // can't determine offset

            int currentClassSuffix = classNameSuffix(classDecl.getNameAsString());
            if (currentClassSuffix < 0) return;

            int offset = baseClassSuffix - tNumber(growingTip.isEmpty() ? "T0" :
                    (fixedArgs.isEmpty() && growingTip.size() == 1 ? growingTip.get(0) : growingTip.get(0)));
            // offset = baseClassSuffix - (currentClassSuffix + 1) ... this gets complex
            // Let's just use: new class suffix = currentClassSuffix + (baseClassSuffix - templateClassSuffix_at_template)
            // We approximate: newSuffix = currentClassSuffix + (baseClassSuffix - currentClassSuffix - 1 + 1) = baseClassSuffix
            // i.e., the return class suffix = current suffix + 1 (always)
            int newSuffix = currentClassSuffix + 1;
            String baseClassName = stripNumericSuffix(info.baseClass());
            String newBaseClass = baseClassName + newSuffix;

            // Check boundary: is the new return type class in the generated set?
            if (!allGeneratedNames.contains(newBaseClass)) {
                toRemove.add(method);
                return;
            }

            // Build new type args: fixed + growing tip up to T(currentClassSuffix + 1)
            List<String> newTypeArgs = new ArrayList<>(fixedArgs);
            // Growing tip at current i: from T(fixedArgs.size()+1) to T(currentClassSuffix+1)
            // declaredTypeParams at current i covers T1..T(currentClassSuffix) after G1 expansion
            // So growing tip = [T(currentClassSuffix+1)]
            for (int j = fixedArgs.size() + 1; j <= newSuffix; j++) {
                newTypeArgs.add("T" + j);
            }

            String newReturnType = newBaseClass + (newTypeArgs.isEmpty() ? "" :
                    "<" + String.join(", ", newTypeArgs) + ">");

            try {
                method.setType(StaticJavaParser.parseType(newReturnType));
            } catch (Exception ignored) { return; }

            // Parameter type inference: update params whose type contains undeclared T+number vars
            // The undeclared var at template was the last growing tip element (e.g. T2 in Step1<T1>)
            // At current i, it maps to T(currentClassSuffix+1)
            if (!growingTip.isEmpty()) {
                String templateTipVar = growingTip.get(growingTip.size() - 1); // e.g. "T2"
                String currentTipVar = "T" + newSuffix; // e.g. "T3" at i=2
                method.getParameters().forEach(param -> {
                    String paramType = param.getTypeAsString();
                    if (paramType.contains(templateTipVar) && !declaredTypeParams.contains(templateTipVar)) {
                        // Replace templateTipVar with currentTipVar in the param type
                        String newParamType = replaceTypeVar(paramType, templateTipVar, currentTipVar);
                        try {
                            param.setType(StaticJavaParser.parseType(newParamType));
                        } catch (Exception ignored) { }
                    }
                });
            }
        });

        toRemove.forEach(m -> classDecl.getMembers().removeIf(member -> member == m));
    }

    private record ReturnTypeInfo(String baseClass, List<String> typeArgs) {}

    /** Parses "Step2<T1, T2>" into ReturnTypeInfo("Step2", ["T1","T2"]). Returns null if unparseable. */
    private static ReturnTypeInfo parseReturnTypeInfo(String returnType) {
        int lt = returnType.indexOf('<');
        if (lt < 0) return new ReturnTypeInfo(returnType.trim(), List.of());
        String base = returnType.substring(0, lt).trim();
        String argsStr = returnType.substring(lt + 1, returnType.lastIndexOf('>')).trim();
        if (argsStr.isEmpty()) return new ReturnTypeInfo(base, List.of());
        // Split on comma at depth 0 (handles nested generics)
        List<String> args = new ArrayList<>();
        int depth = 0;
        int start = 0;
        for (int i = 0; i < argsStr.length(); i++) {
            char c = argsStr.charAt(i);
            if (c == '<') depth++;
            else if (c == '>') depth--;
            else if (c == ',' && depth == 0) {
                args.add(argsStr.substring(start, i).trim());
                start = i + 1;
            }
        }
        args.add(argsStr.substring(start).trim());
        return new ReturnTypeInfo(base, args);
    }

    /** Returns true if {@code s} matches the T+number pattern (e.g. "T1", "T23"). */
    private static boolean isTNumberVar(String s) {
        if (s.length() < 2 || s.charAt(0) != 'T') return false;
        for (int i = 1; i < s.length(); i++) {
            if (!Character.isDigit(s.charAt(i))) return false;
        }
        return true;
    }

    /** Extracts the number from a T+number variable (e.g. "T3" → 3). Returns -1 if not T+number. */
    private static int tNumber(String s) {
        if (!isTNumberVar(s)) return -1;
        try { return Integer.parseInt(s.substring(1)); } catch (Exception ignored) { return -1; }
    }

    /** Extracts the numeric suffix from a class name (e.g. "Step3" → 3, "Step" → -1). */
    private static int classNameSuffix(String name) {
        int i = name.length() - 1;
        if (i < 0 || !Character.isDigit(name.charAt(i))) return -1;
        while (i > 0 && Character.isDigit(name.charAt(i - 1))) i--;
        try { return Integer.parseInt(name.substring(i)); } catch (Exception ignored) { return -1; }
    }

    /** Strips the numeric suffix from a class name (e.g. "Step3" → "Step"). */
    private static String stripNumericSuffix(String name) {
        int i = name.length() - 1;
        if (i < 0 || !Character.isDigit(name.charAt(i))) return name;
        while (i > 0 && Character.isDigit(name.charAt(i - 1))) i--;
        return name.substring(0, i);
    }

    /** Replaces all word-boundary occurrences of {@code oldVar} with {@code newVar} in {@code type}. */
    private static String replaceTypeVar(String type, String oldVar, String newVar) {
        // Use regex word boundary to avoid replacing "T1" in "T10"
        return type.replaceAll("(?<![\\w])\\Q" + oldVar + "\\E(?![\\w])", newVar);
    }
