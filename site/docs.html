Step by Step

Quick Start

Step 1 — Add the processor

<annotationProcessorPaths>
  <path>
    <groupId>io.quarkiverse.permuplate</groupId>
    <artifactId>quarkus-permuplate-processor</artifactId>
    <version>1.0-SNAPSHOT</version>
  </path>
  <path>
    <groupId>com.github.javaparser</groupId>
    <artifactId>javaparser-core</artifactId>
    <version>3.25.9</version>
  </path>
  <path>
    <groupId>org.apache.commons</groupId>
    <artifactId>commons-jexl3</artifactId>
    <version>3.3</version>
  </path>
</annotationProcessorPaths>

Step 2 — Write a template

@Permute(varName = "i",
         from = "3", to = "10",
         className = "Join${i}")
public class Join2 {

    private @PermuteDeclr(
        type = "Callable${i}",
        name = "c${i}") Callable2 c2;

    public void join(
        @PermuteParam(varName = "j",
            from = "1", to = "${i-1}",
            type = "Object",
            name = "o${j}") Object o1) {
        c2.call(o1, o2);
    }
}

Step 3 — Run mvn compile

mvn compile
# Join3.java, Join4.java ... Join10.java appear in target/generated-sources/annotations/
# No scripts. No committed generated files. Just javac.
Reference

Annotation Reference

Annotation
Placed on
What it does
@Permute
class / method
Outer loop. Generates N types or collects N method overloads into one class. Supports from/to as JEXL expressions, string constants, cross-product via extraVars, and inline mode via Maven plugin.
@PermuteVar
nested on @Permute
Additional integer loop axis. One class per combination of primary variable and all extra variables — cross-product generation with a single template.
@PermuteDeclr
field / param / for-each
Scope-aware rename: field → class-wide, constructor param → constructor body, for-each variable → loop body only. name="" changes type only (method params).
@PermuteParam
method / constructor param
Sentinel expansion. Inner loop generates o1..o(i-1). Call-site anchor expansion rewrites every call in the body — including inside lambdas, with proper scoping.
@PermuteTypeParam
type parameter
Expands a sentinel type parameter. Implicit when @PermuteParam uses T${j} naming. Bounds propagated: T1 extends Comparable<T1> → each generated Tk extends Comparable<Tk>.
@PermuteConst
field / local var
Replaces initializer with evaluated JEXL expression. Combinable with @PermuteDeclr on the same field — rename first, substitute value second.
@PermuteValue
field / var / method
Superset of @PermuteConst. Also replaces the RHS of the assignment statement at index (0-based) in a method or constructor body.
@PermuteReturn
method
Controls return type per permutation. Methods are silently omitted when the evaluated return type is not in the generated set — the leaf-class pattern. when="true" overrides.
@PermuteMethod
method
Inner loop generates one overload per value. to defaults to @Permute.to - i (inferred). Empty range = leaf class; method omitted silently.
@PermuteExtends
class
Explicit extends/implements clause control. Implicit same-N expansion fires automatically for standard naming — use this only when inference doesn't apply. Maven plugin inline mode only.
@PermuteCase
method / constructor
Inserts switch cases from an inner loop. Seed case and default preserved. New cases inserted before default. Empty range = no cases inserted.
@PermuteStatements
method / constructor
Inserts generated statements at position="first" or position="last". Applied after @PermuteValue.
@PermuteImport
class (repeatable)
Adds a JEXL-evaluated import to each generated class. Stripped from output. Use when generated types reference classes the template's own imports can't cover.
Expressions

Built-in Functions

All annotation string attributes are evaluated by Apache Commons JEXL3. These built-in functions are available in every ${...} expression without any prefix or import.

alpha(n)

Integer → uppercase letter, 1-indexed. 1=A, 26=Z. Out of range throws at generation time.

${alpha(j)} → A, B, C
lower(n)

Integer → lowercase letter, 1-indexed. 1=a, 26=z. Same range as alpha.

${lower(j)} → a, b, c
typeArgList(from, to, style)

Comma-separated type argument list. Styles: "T", "alpha", "lower". Returns "" when from > to.

${typeArgList(1,i,'T')}
→ T1, T2, T3
